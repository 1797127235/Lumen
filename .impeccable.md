# Design Context · CodePilot 前端

> 这是 `/impeccable:impeccable` 的项目级设计上下文。后续每次设计/前端工作前会先读这个文件。
> 对应产品：`CodePilot 码路领航` — FastAPI + LangGraph 的 AI 学长，陪计算机专业学生从大一走到拿 Offer。

---

## Users · 谁在用

主要受众：**国内计算机专业在校生（大一—研三）**,6 类典型画像：

1. 大一探索者 — 刚进专业，对方向迷茫，需要被带着看世界
2. 大二/三焦虑追赶者 — GPA 普通，怕被同龄人甩开，需要清晰路线
3. 转码同学 — 跨专业进来，缺基础也缺归属感
4. 大四求职党 — 简历改了 N 版，面试碰壁
5. 研究生 — 在科研与就业之间撕扯
6. 多 Offer 选择者 — 信息过载，做不了决策

**使用场景的物理画面**：宿舍/自习室深夜、笔记本屏幕、戴着耳机、有点焦虑、刷题/写代码累了的间歇 — 不是上班族盯着仪表盘。

**他们要做什么 (jobs-to-be-done)**：
- 对话问"我该走哪个方向" — 需要被认真听+给方向
- 上传简历 → 看到自己的画像 — 需要被准确"看见"
- 粘贴 JD → 看到差距 — 需要客观+不打击

## Brand Personality · 三个词

**温柔的学长 · 手写感 · 偏人文**

(Gentle senior · Handwritten · Humanistic)

**核心比喻**：这个界面不是 dashboard,是**学长摊在你桌上的一本笔记**。
他认真听过你的问题,写下来,递给你。字是手写的,纸是暖的,灯是台灯。

**学长口吻**(来自 `docs/功能模块与交互设计…md` 6.4 节)：
- 不装腔("作为一个 AI 模型…" 不允许)
- 有据可查(不空喊"加油",给具体步骤)
- 共情但不腻(节制的温度,不滥用 emoji)
- 第一人称"我",第二人称"你",避免"用户"

## Aesthetic Direction · 美学方向

**主题**：**仅深色** — 推断自使用场景(深夜+疲惫+长时间阅读)。

**三个明确反对**：

1. **不要"科技蓝+活力橙"的招聘网站审美** — 即使产品文档建议过,照做就是 AI slop。
2. **不要"霓虹紫蓝渐变+发光边框"的 AI 工具审美** — ChatGPT、Cursor、各种 AI 套壳已用烂。
3. **不要 dashboard 化** — 大数字、小标签、柱状图、卡片网格。这不是数据产品。

**核心隐喻**：纸 / 笔 / 灯 / 笔记。
颜色取自暖色台灯下的旧本子,不是 RGB 屏幕。

## Design Principles · 派生原则

1. **Warmth comes from typography and copy, not from gradients.**
   暖意来自字形(手写楷体)和文案(学长口吻),不来自渐变发光。

2. **It's a notebook, not a dashboard.**
   信息密度和视觉重量都偏低。可以留白,可以缓慢。

3. **Single accent, used like ink.**
   一个暖色重点色(琥珀/纸金),用得越少越有力量。不要彩色铺满。

4. **Chinese is first-class, not afterthought.**
   字体配对、行高、字号都按中文为主、英文为辅来调(不是反过来)。

5. **No AI tells.**
   禁用 impeccable skill 列出的反模式：左/右 stripe border、渐变文字、glassmorphism 满屏、sparkline 装饰、AI 紫蓝色板。

## 已确定的技术栈

- **栈**：React + Vite + TypeScript
- **目录**：`app/frontend/`(Vite dev server 通过 proxy 转发 `/api` → FastAPI:8000)
- **UI 库**：Tailwind + shadcn/ui(按需挑选,深度改造样式)
- **API 契约**：见 `README.md` API 章节(chat / profile / jd / health 四组)
- **认证**：Phase 1 用 localStorage 持久化的 UUID 作为 user_id(无 JWT)

具体字体/调色板/IA 见 `docs/frontend-design.md`。
