# 简历智能优化

> 源自：功能模块与交互设计_AI职业规划智能体.md

### 核心功能④：简历智能解析与优化

#### 功能定位与价值主张

**定位**: 将"一份简历投所有岗位"升级为"千人千面的精准投递"，通过JD匹配度分析、STAR法则优化、关键词补充，显著提升简历过筛率。

**价值主张**: "不只是改病句，而是帮你想清楚——这段经历想证明什么能力？面试官看到这句话会问什么？" — 从面试官视角重写简历。

#### 用户触发条件

| 触发场景 | 用户操作 |
|---------|---------|
| 准备投递 | 上传现有简历 + 粘贴目标岗位JD |
| 面试挂掉怀疑简历 | 上传简历询问"简历有什么问题" |
| 方向切换 | 已有简历但想换方向投递 |
| 定期迭代 | 每两周优化一次简历版本 |

#### 输入与输出定义

| 维度 | 定义 |
|------|------|
| **用户输入** | 简历文件（PDF/Word/图片）+ 目标岗位JD（文本） |
| **系统输出** | ① 结构化简历信息提取 ② JD匹配度评分 ③ 关键词缺失分析 ④ 逐条优化建议 ⑤ 优化后简历预览 ⑥ ATS友好性检测报告 |
| **输出格式** | 左右分栏：左侧原始简历预览，右侧分析报告 + 优化建议 |

#### 详细交互流程

**步骤1 - 简历上传**
- 用户操作：拖拽或点击上传简历文件，支持PDF/Word/PNG/JPG
- 系统响应：显示上传进度，完成后"正在解析简历..."（OCR+NLP提取）
- 解析结果展示：
  ```
  【解析结果确认】请检查识别是否准确：
  姓名：张三 ✓
  学校：XX大学 ✓
  专业：软件工程 ✓
  电话：138****8888 ✓
  经历1：XX公司实习 - 后端开发（2024.06-2024.09）✓
  项目1：电商系统（Spring Boot+MySQL）
  ...
  [有错误？手动修正]
  ```

**步骤2 - JD粘贴/选择**
- 用户操作：粘贴目标岗位JD文本，或从收藏夹选择已保存的JD
- 系统操作：解析JD提取核心要求（技能关键词、经验要求、软技能）

**步骤3 - 匹配度分析**
- 系统输出：
  ```
  【匹配度评分】67/100 — 有提升空间
  
  [匹配热力图]
  Java基础 ████████████ 90% ✅
  Spring框架 ████████░░ 65% ⚠️
  数据库优化 ████░░░░░░ 40% 🔴 缺失
  分布式经验 █░░░░░░░░░ 10% 🔴 严重缺失
  团队协作 █████████░░ 80% ✅
  
  【关键词缺失】以下JD关键词未在你的简历中体现：
  • "微服务架构" -> 建议补充位置：项目描述
  • "Redis缓存" -> 建议补充位置：技能列表/项目描述
  • "高并发" -> 建议补充位置：项目描述（量化指标）
  ```

**步骤4 - 逐条优化建议**
- 系统操作：遍历简历各模块，给出具体修改建议
  ```
  【教育背景】✅ 基本完整，建议补充：
    -> " GPA 3.6/4.0（前15%）" 如成绩好可加上
  
  【实习经历】⚠️ 需要优化：
    原文："负责后端模块开发，参与需求评审和代码编写"
    问题：过于笼统，无技术深度，无量化成果
    建议改为：
    "负责订单系统核心模块开发，使用Spring Boot + MyBatis实现订单状态机流转，
     设计并接入Redis缓存降低数据库查询量40%，参与Code Review提升团队代码规范覆盖率至90%"
    [一键应用此建议] [查看为什么这么改]
  
  【项目经历】🔴 严重需要优化：
    原文："做了一个电商系统，实现了用户登录、商品展示、购物车功能"
    问题：① "做了一个"显得随意 ② 纯功能罗列 ③ 无技术难点和成果
    建议改为：
    "独立设计并实现高并发电商秒杀系统，通过Redis预减库存+消息队列异步下单+数据库乐观锁，
     支撑1000 QPS并发场景下库存超卖率降至0，项目获课程设计优秀（前5%）"
    [一键应用此建议] [查看为什么这么改]
  ```

**步骤5 - 一键优化与预览**
- 用户操作：可选择"一键应用全部建议"或逐条确认
- 系统输出：优化后的简历预览，标注所有修改点（删除线+高亮）
- 提供多模板切换：大厂风格（简洁、关键词突出）/ 外企风格（STAR详细）/ 国企风格（稳重、全面）

**步骤6 - ATS检测与导出**
- 系统操作：模拟ATS系统扫描，检测：
  - 关键词密度是否合理（避免堆砌）
  - 格式是否会被解析错误（表格/图片/复杂排版）
  - 文件大小是否合适
- 用户操作：下载优化后的PDF/Word，或保存到【个人数据中心-求职档案】
- 版本管理：自动命名"张三_后端开发_字节跳动_20250601_v2"

#### 边界情况与异常处理

| 边界情况 | 处理策略 |
|---------|---------|
| 简历解析失败（扫描件质量差） | "简历图片清晰度不够，部分文字识别困难。建议重新上传或手动输入关键信息。" |
| 用户无实习/项目经历 | "目前经历较薄，建议：① 先使用【项目实践指导】生成一个可执行项目 ② 将课程设计/实验包装为项目" |
| JD与简历方向完全不匹配 | "这份JD是前端岗位，你的简历偏向后端，建议先确定方向再优化，或生成一份前端版本的简历" |
| 优化后简历超过1页（应届生） | 提示"应届生简历建议控制在1页，以下信息可考虑精简：[列出低优先级内容]" |
| 用户频繁优化同一简历 | 显示版本对比"本次相比v2改了XX处，整体匹配度从62%提升到71%" |

#### AI Agent角色与工作流

**Agent角色**: "简历顾问Agent"
- 人设：大厂HR+技术面试官双重视角，知道HR如何筛简历、面试官如何看项目
- 核心能力：简历解析 + JD语义匹配 + STAR法则重写 + ATS规则检测

**工作流**:
```
简历上传 -> OCR/NLP解析 -> 结构化信息提取 -> 
JD粘贴 -> 关键词提取 -> 
匹配度计算（TF-IDF + 语义相似度） -> 
逐模块诊断（问题识别） -> 
STAR法则优化生成（做了什么->用什么技术->遇到什么难点->如何解决->取得什么效果） -> 
ATS规则检测 -> 输出优化简历
```
