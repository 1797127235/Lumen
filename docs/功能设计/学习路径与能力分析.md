# 学习路径与能力分析

> 源自：功能模块与交互设计_AI职业规划智能体.md

### 核心功能②：个性化学习路径生成

#### 功能定位与价值主张

**定位**: 将"碎片化收藏"转化为"可执行路线"，基于用户目标岗位、当前基础、可用时间，生成定制化的渐进式学习路径。

**价值主张**: "不是给你一张地图，而是给你一张根据你的体能和目标定制的训练计划" — 标注每个阶段学什么、学到什么程度、用多久、如何验收。

#### 用户触发条件

| 触发场景 | 用户操作 |
|---------|---------|
| 刚确定方向 | 在咨询对话中或画像设置里选择了"Java后端"方向 |
| 需要系统化 | 收藏了50篇博客但从未执行，主动寻求路径规划 |
| 目标明确 | 输入目标公司+岗位（如"字节跳动后端开发"） |
| 路径调整 | 已生成路径但想切换方向或调整难度 |

#### 输入与输出定义

| 维度 | 定义 |
|------|------|
| **用户输入** | 目标岗位（如"后端开发"）、当前基础（多选：零基础/学过C/做过项目）、可用时间（每天1h/2h/4h）、院校层级（985/211/双非）、目标公司层级（大厂/中厂/国企）、截止时间（如"大三暑假实习前"） |
| **系统输出** | 结构化学习路径，包含：①总览时间线（甘特图形式） ②阶段列表（每个阶段含节点名称、预计时长、验收标准、推荐资源） ③里程碑标记点 ④风险提醒（如"此阶段耗时较长，注意保持节奏"） |
| **输出格式** | 可视化时间线 + 可展开的卡片列表 + 一键导出日历/待办 |

#### 详细交互流程

**步骤1 - 信息收集（对话式表单）**
- 系统操作：以对话方式逐条询问关键参数，避免表单疲劳
  ```
  AI: "想给你定制一条学习路线，先聊几句~ 你目标的方向是什么？"
  选项卡：[Java后端] [Go后端] [前端] [算法] [大数据] [其他]
  
  用户: [点击 Java后端]
  
  AI: "好嘞！Java后端是个经典方向。你现在基础怎么样？"
  选项卡：[零基础，只学过C语法] [学过Java基础语法] [做过小项目] [较熟练，想冲大厂]
  ```
- 边界处理：用户不确定目标时，提供"帮我推荐"按钮，跳转方向测评

**步骤2 - 参数确认与路径计算**
- 系统操作：
  1. 将用户输入标准化为路径生成参数
  2. 调用路径规划引擎（基于岗位技能图谱 + 渐进式依赖图）
  3. 计算总时长、阶段拆分、资源匹配
- 系统响应：显示"正在为你规划路线..."进度动画（预计3-5秒）

**步骤3 - 路径可视化展示**
- 系统输出：顶部时间线（横向滚动）+ 下方详细卡片
  ```
  ━━━ 第1阶段：Java基础夯实（4周）━━━ 第2阶段：数据库核心（3周）━━━ ...
  
  [阶段卡片]
  阶段1：Java SE基础 -> 面向对象 -> 集合框架 -> 异常处理 -> IO流
  预计时长：4周（每天2小时）
  验收标准：能独立手写一个带连接池的JDBC工具类，理解多线程基础
  推荐资源：
    - 书籍：《Java核心技术 卷I》第3-8章
    - 视频：韩顺平Java（B站免费）
    - 练习：LeetCode简单题10道
  里程碑测验：[开始测验]（5道选择题+1道编程题）
  ```

**步骤4 - 用户确认与调整**
- 用户操作：可拖拽调整阶段顺序（系统校验依赖关系，如不能先学Spring再学Java基础），或点击"太紧了/太松了"调整每日时长
- 系统响应：实时重新计算并展示调整后的时间线

**步骤5 - 路径保存与日程同步**
- 用户操作：点击"保存到我的计划"，选择是否同步到系统日历/手机日历
- 系统操作：
  1. 保存路径到用户学习档案
  2. 生成每周学习任务提醒（如"本周目标：完成MySQL索引章节"）
  3. 在【个人数据中心-学习档案】创建条目

**步骤6 - 进度追踪入口**
- 系统操作：在学习路径页面常驻"标记进度"按钮，用户完成节点后打勾
- 联动效果：已完成的节点变绿，进行中的节点闪烁提示，未开始的节点置灰

#### 边界情况与异常处理

| 边界情况 | 处理策略 |
|---------|---------|
| 用户目标岗位不在库中 | "这个方向目前数据较少，我先按'后端通用'给你规划，其中XX部分你可以替换成目标技术" |
| 截止时间过紧（如2个月要掌握6个月内容） | 红色警告"时间非常紧张，建议：①延长截止时间 ②降低目标公司层级 ③聚焦核心技能放弃加分项" |
| 用户基础与目标跨度太大 | 弹出建议"你的基础和目标差距较大，建议分两步走：第一阶段打基础（6个月），第二阶段冲刺（3个月）" |
| 依赖冲突（用户调整顺序违反依赖） | 阻止调整并提示"Spring Boot需要先掌握Java基础，建议先完成阶段1" |

#### AI Agent角色与工作流

**Agent角色**: "学习规划Agent"
- 人设：刚拿到大厂offer的学长，擅长拆解目标、制定计划
- 核心能力：技能图谱检索 + 渐进式依赖计算 + 资源匹配

**工作流**:
```
参数收集 -> 目标岗位技能图谱查询 -> 当前基础匹配 -> 
缺失技能排序（按面试频率和依赖关系） -> 
渐进式路径编排（确保前置技能先学） -> 
学习资源匹配（书籍/视频/文档/练习题） -> 
时间分配计算 -> 生成可视化路径
```

### 核心功能③：岗位能力差距分析

#### 功能定位与价值主张

**定位**: 将"模糊的焦虑"转化为"清晰的行动清单"，客观评估用户当前技能水平与目标岗位要求的具体差距。

**价值主张**: "不是告诉你'你还差得远'，而是精确到'Redis持久化机制这道面试题你还答不上来，需要补这里'" — 用数据将焦虑转化为可执行的行动。

#### 用户触发条件

| 触发场景 | 用户操作 |
|---------|---------|
| 看到JD后自我怀疑 | 粘贴一份JD，让系统分析自己与岗位的匹配度 |
| 定期体检 | 每月主动进行一次能力盘点 |
| 路径生成后 | 学习路径中某阶段完成后进行自评 |
| 面试挂掉后 | 输入挂掉的面试环节，系统分析短板 |

#### 输入与输出定义

| 维度 | 定义 |
|------|------|
| **用户输入** | 目标岗位JD（文本粘贴）或选择预设岗位模板；用户自评（1-5分）或选择"通过测试评估" |
| **系统输出** | ① 岗位能力模型拆解（维度列表） ② 能力雷达图（当前vs目标） ③ 差距等级标注（达标/接近/差距大/缺失） ④ 补全建议（优先级排序） ⑤ 预计补全时长 |
| **输出格式** | 雷达图可视化 + 可排序的维度卡片 + 一键生成补全学习路径 |

#### 详细交互流程

**步骤1 - 目标岗位设定**
- 用户操作：
  - 方式A：粘贴一份JD文本
  - 方式B：选择预设模板（如"字节跳动-后端开发-2-1""阿里巴巴-Java开发-P5"）
  - 方式C：从已生成的学习路径直接关联目标岗位
- 系统操作：解析JD，提取关键技能要求（如"熟悉Redis"->拆解为"数据结构/持久化/集群/性能优化"）

**步骤2 - 评估方式选择**
- 系统操作：提供两种评估方式
  ```
  "想怎么评估？"
  [快速自评] 我对每个技能点自己打分（2分钟）
  [深度测评] 做题+问答，系统帮我评估（15-20分钟）
  ```
- 边界处理：首次使用建议深度测评，后续可用快速自评

**步骤3 - 深度测评流程（选择深度测评时）**
- 系统操作：按能力维度轮流出题
  ```
  【技术维度：Redis】
  Q1: Redis支持哪些数据结构？（多选）
  Q2: 场景题：如果Redis突然变慢，你的排查思路是什么？（简答，AI评分）
  Q3: 编程题：用Redis实现一个分布式锁。（在线编码）
  ```
- 每题完成后：即时反馈正确/部分正确/错误，附解析
- 全部完成后：显示"分析中..."，计算各维度得分

**步骤4 - 生成差距分析报告**
- 系统输出：
  ```
  【岗位：字节跳动-后端开发】能力差距分析报告
  
  [雷达图] 你的得分 vs 岗位要求
  
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  【达标】Java基础（85/100）✅
    -> 保持即可，面试前复习集合框架源码
  
  【接近】MySQL（62/100）⚠️ 需提升
    -> 差距点：索引优化、事务隔离级别、慢查询分析
    -> 建议学习：《MySQL实战45讲》第15-22讲
    -> 预计补全：2周
  
  【差距大】Redis（35/100）🔴 重点突破
    -> 差距点：持久化机制(RDB/AOF)、集群模式、缓存击穿/穿透/雪崩
    -> 建议学习：路径已生成 [查看补全路径]
    -> 预计补全：3周
  
  【缺失】分布式系统（0/100）❌ 从零开始
    -> 差距点：CAP理论、一致性协议、微服务架构
    -> 建议学习：路径已生成 [查看补全路径]
    -> 预计补全：6周
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  
  综合评估：当前匹配度 47%，预计11周可达80%（达到面试门槛）
  [生成补全学习路径] [保存报告] [分享报告]
  ```

**步骤5 - 补全路径生成**
- 用户操作：点击"生成补全学习路径"
- 系统操作：将差距维度按优先级排序（面试频率高->前置依赖->难度递增），生成针对性补全路径
- 联动：自动与现有学习路径合并，标注新增节点

**步骤6 - 定期复盘提醒**
- 系统操作：保存报告时询问"是否每月自动复测？"
- 用户确认后：每月推送复测提醒，对比历史雷达图展示进步曲线

#### 边界情况与异常处理

| 边界情况 | 处理策略 |
|---------|---------|
| JD解析失败（格式混乱） | "这份JD格式比较特别，我提取到以下关键要求：[列出]，如果遗漏了你手动补充一下？" |
| 用户自评明显偏高/偏低 | 与历史测评数据对比，偏差大时提示"你自评的Java基础和上次测试差距较大，建议用测评验证一下" |
| 目标岗位要求超出学生合理范围 | "这个岗位是社招P6级别，对在校生要求过高，建议换成'字节后端实习'或'校招P5'作为目标" |
| 测评中途退出 | 自动保存进度，下次进入提示"上次测评做到Redis部分，继续吗？" |

#### AI Agent角色与工作流

**Agent角色**: "能力诊断Agent"
- 人设：技术面试官视角，严格但公正，善于发现盲点
- 核心能力：JD解析 + 能力模型拆解 + 多维度测评 + 差距量化

**工作流**:
```
JD输入 -> JD解析引擎（提取技能关键词） -> 
岗位能力模型查询（技能->知识点映射） -> 
评估方式选择 -> 出题/自评收集 -> 
多维度评分计算 -> 差距等级判定 -> 
补全优先级排序（面试频率 x 提升空间 x 依赖关系） -> 
生成差距报告 + 补全路径
```
