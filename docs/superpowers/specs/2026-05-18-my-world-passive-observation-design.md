# Design: 我的世界 — 被动观察层

Generated by /superpowers:brainstorming on 2026-05-18
Branch: master
Status: DRAFT

---

## 问题陈述

"我的世界"模块目前只支持本地文件夹（`local_folder`）。Lumen 只能读到用户主动写下来的东西，但用户真正的注意力——他们在读什么、什么触动了他们——完全不可见。

**核心缺口：** Lumen 知道你说了什么，但不知道你在关注什么。这两者之间往往有巨大的落差，而这个落差本身就是最有价值的洞察。

**目标：** 让 Lumen 被动感知用户的真实注意力，不要求用户做任何额外的行为。

---

## 数据源信号质量分析

| 数据源 | 谁在主动 | 信号质量 | 原因 |
|--------|---------|---------|------|
| 本地文件夹（已有） | 用户主动写 | 中 | 你写的，但不一定是最在意的 |
| 手动 URL | 用户主动发 | 中 | 觉得值得分享的 |
| 书签同步 | 用户主动存 | 中低 | 存了就忘的比例高 |
| **浏览器阅读行为** | **被动捕获** | **高** | 真正花时间的 |
| **Readwise 高亮** | 用户主动划 | **最高** | 触动你的原话 |

---

## 优先级

```
P0  浏览器阅读行为（Chrome 插件）   被动，高质量信号
P0  Readwise 高亮同步               主动但极精准
P1  Obsidian 专属优化               识别双链、tags、frontmatter
P2  其他数据源                      待后续评估
```

### 数据流

```
用户在 Chrome 浏览页面
        │
        ▼
content.js 采集行为信号
（visibilityState + 滚动 + 选中文字 + 活跃时间）
        │
        ▼
background.js 计算兴趣评分
        │ score > 阈值
        ▼
POST http://localhost:8000/api/data-sources/browser/ingest
        │
        ▼
BrowserActivityConnector.receive()
        │
        ▼
现有 Pipeline（解析 → 向量化 → 存储）
```

## 浏览器插件设计

### 兴趣评分算法

```javascript
// content.js

const PAGE_WORD_COUNT = document.body.innerText.split(/\s+/).length
const EXPECTED_READ_TIME = PAGE_WORD_COUNT / 200  // 分钟，200wpm

let focusedSeconds = 0
let lastActiveTime = Date.now()
let isVisible = true

// 只计算真正在看的时间（标签页在前台 + 有活动）
document.addEventListener('visibilitychange', () => {
  isVisible = document.visibilityState === 'visible'
})

setInterval(() => {
  if (isVisible && hasRecentActivity()) {
    focusedSeconds++
  }
}, 1000)

// 文字选中 —— 最强信号
const selectedTexts = []
document.addEventListener('mouseup', () => {
  const selection = window.getSelection().toString().trim()
  if (selection.length > 20) {
    selectedTexts.push(selection)
  }
})

// 离开页面时计算评分
window.addEventListener('beforeunload', () => {
  const readingRatio = (focusedSeconds / 60) / EXPECTED_READ_TIME
  const score = calculateScore({ readingRatio, selectedTexts, returnVisits })
  if (score > 0.3) sendToLumen({ url, title, score, selectedTexts, readingRatio })
})
```

### 兴趣评分权重（初始假设，需迭代）

> **注意：** 以下权重是初始启发式假设，不是最终设计。需要在真实使用中收集数据后校准。

| 信号 | 规则 | 初始权重 |
|------|------|------|
| 文字选中 | 每段 +0.3，最多 +0.6 | 最高 |
| 阅读比率 > 0.5 | 读了一半以上 | 中高 |
| 回访 | 关掉又打开 | 中 |
| 阅读比率 0.2-0.5 | 扫读 | 低 |
| 停留 < 10 秒 | 直接丢弃，不发送 | — |

**评估方式：**
- 插件在本地记录每条发送记录（url、score、各信号值）到 `~/.lumen/browser-activity.jsonl`
- 用户可在"我的世界"页面看到"最近被记录的页面"列表，标记"不应该被记录"
- 收集 1-2 周真实数据后，根据误判比例调整各信号权重
- 目标：误判率（不应记录但记录了）< 10%

### 发送给 Lumen 的数据

```json
{
  "url": "https://...",
  "title": "文章标题",
  "interest_score": 0.81,
  "reading_ratio": 0.73,
  "focused_minutes": 4.2,
  "selected_texts": ["你主动选中的段落..."],
  "visited_at": "2026-05-18T10:30:00Z",
  "word_count": 1840
}
```

**注意：** 插件本身不抓取页面正文。正文由后端的 `BrowserActivityConnector` 根据 URL 请求，仅当 `interest_score > 0.5` 时才抓取，减少不必要的网络请求。

## BrowserActivityConnector

继承现有 `DataSourceConnector`，实现 HTTP 接收而非文件扫描：

```python
class BrowserActivityConnector(DataSourceConnector):
    """接收浏览器插件推送的阅读行为数据。"""

    _source_id = "browser_activity"

    async def receive(self, payload: BrowserActivityPayload) -> RawBytes | None:
        """处理单条浏览行为推送。"""
        if payload.interest_score < 0.3:
            return None

        content = ""
        if payload.selected_texts:
            # 选中文字优先，质量最高
            content = "\n\n".join(payload.selected_texts)
        elif payload.interest_score > 0.5:
            # 分数够高才抓正文
            content = await self._fetch_page_content(payload.url)

        if not content:
            return None

        return RawBytes(
            data_source_id=self._data_source_id,
            external_id=payload.url,
            uri=payload.url,
            content_bytes=content.encode(),
            mime_type="text/html",
            metadata={
                "title": payload.title,
                "interest_score": payload.interest_score,
                "reading_ratio": payload.reading_ratio,
                "selected_texts": payload.selected_texts,
                "source": "browser_activity",
            },
            last_modified=payload.visited_at.timestamp(),
            connector_type="browser_activity",
        )

    async def scan(self) -> AsyncIterator[RawBytes]:
        # 浏览器行为是推送模式，不支持全量扫描
        return
        yield

    def start_watching(self, ...):
        pass  # 插件推送，无需 watch

    def stop_watching(self):
        pass
```

新增路由 `POST /api/data-sources/browser/ingest` 接收插件推送，直接调用 `BrowserActivityConnector.receive()`。

## Readwise 连接器

```python
class ReadwiseConnector(DataSourceConnector):
    """同步 Readwise 高亮。"""

    _source_id = "readwise"
    API_BASE = "https://readwise.io/api/v2"

    async def scan(self) -> AsyncIterator[RawBytes]:
        """全量同步所有高亮。"""
        async for highlight in self._paginate("/highlights/"):
            doc = self._highlight_to_raw(highlight)
            if doc:
                yield doc

    def _highlight_to_raw(self, h: dict) -> RawBytes | None:
        text = h.get("text", "").strip()
        if not text:
            return None

        content = f"# {h.get('title', '未命名')}\n\n> {text}"
        if note := h.get("note"):
            content += f"\n\n你的笔记：{note}"

        return RawBytes(
            data_source_id=self._data_source_id,
            external_id=f"readwise:{h['id']}",
            uri=h.get("url") or f"readwise:{h['id']}",
            content_bytes=content.encode(),
            mime_type="text/markdown",
            metadata={
                "source": "readwise",
                "book_title": h.get("title"),
                "author": h.get("author"),
                "highlighted_at": h.get("highlighted_at"),
                "tags": h.get("tags", []),
            },
            last_modified=...,
            connector_type="readwise",
        )
```

配置只需一个 API Token（用户在 Readwise 设置页生成）。

## 隐私设计

- 插件默认不发送任何数据，需要用户在弹窗里手动开启
- 评分低于 0.3 的页面在插件端直接丢弃，不发送
- 内网/localhost URL 自动过滤
- 用户可以在"我的世界"页面随时暂停或删除 browser_activity 数据源
- 正文抓取只在 interest_score > 0.5 时触发，不是每个页面都抓

---

## 成功标准

- 用户添加浏览器阅读数据源后，一周内 Lumen 能在对话中引用用户读过但没主动提及的内容
- Readwise 高亮同步后，Lumen 能说出"你在《X》里划了一句话：..."
- interest_score 误判率（用户觉得不应该被记录的内容被记录）< 10%
