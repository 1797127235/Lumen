## 一、执行摘要

### 1.1 项目概述

**产品名称**：「CodePilot · 码路领航」（建议）  
**核心定位**：计算机专业学生的AI职业规划伙伴  
**产品形态**：基于Multi-Agent架构的AI原生职业规划智能体系统  
**目标用户**：计算机相关专业在校生（本科/研究生）、转码学生  
**核心价值**：从入学到就业的全周期职业规划陪伴，解决"方向迷茫、路径不清、求职焦虑"三大痛点

### 1.2 核心数据与机会洞察

| 洞察维度 | 关键数据 | 产品机会 |
|---------|---------|---------|
| 就业困境 | 2024届计算机就业率降至68.65%，年招生43万+，岗位仅30万 | 职业规划从"可选"变为"刚需" |
| 规划空白 | AI求职工具在"职业规划"环节渗透率最低 | 赛道相对蓝海，垂直深耕空间大 |
| 信息鸿沟 | 72%学生认为辅导员不懂技术术语 | AI Agent填补"懂技术+懂职业"的空白 |
| 工具断层 | 牛客/LeetCode只管面试，不管规划 | 构建"规划→学习→求职"全链路闭环 |
| AI优势 | AI Agent可将一次性建议升级为长期陪伴 | 从"问答工具"进化为"数字导师" |

### 1.3 产品一句话

> "一位24小时在线、懂计算机行业、陪你从大一走到毕业的AI学长。"

### 1.4 实施路线总览

| 阶段 | 周期 | 目标 | 核心交付 |
|------|------|------|---------|
| **MVP** | 0-3个月 | 验证核心价值 | 职业咨询 + 学习路径 + 简历优化 + 模拟面试 |
| **增强版** | 4-6个月 | 完善场景闭环 | 进度管理 + Offer决策 + 动态调优 + 成就体系 |
| **生态版** | 7-12个月 | 构建用户生态 | 社区 + 导师对接 + 高校SaaS + 多模态 |

## 二、目标用户与需求场景

### 2.1 六类核心用户画像

| 用户画像 | 阶段 | 核心痛点 | 系统价值 |
|---------|------|---------|---------|
| **迷茫的探索者**（大一新生） | 入学适应期 | 信息过载、方向认知盲区、自我认知模糊 | 方向导览 + 智能测评 + 四年规划 |
| **焦虑的追赶者**（大二/大三） | 技能积累期 | 学习路径碎片化、知识浅层化、缺乏反馈 | 定制学习路径 + 差距诊断 + 进度追踪 |

## 三、竞争策略与差异化定位

### 3.1 竞品格局全景

市场存在12+直接竞品和5+间接替代品，但**职业规划环节存在明显空白**：

- **LoopCV/Kickresume**：通用型AI求职工具，缺乏计算机专业深度
- **牛客网/LeetCode**：最强IT求职社区/刷题平台，但职业规划功能几乎为零
- **Roadmap.sh**：静态学习路线图，无个性化
- **超级简历**：简历工具为主，职业规划是附属功能
- **面试狗/面霸**：存在伦理争议的面试作弊工具
- **通用大模型（ChatGPT等）**：缺乏长期记忆、垂直深度和主动陪伴能力

### 3.2 核心差异化

| 竞品定位 | 我们的差异 |
|---------|-----------|
| 简历工具 | **规划前置**：在积累经历之前就规划"该积累什么" |
| 刷题平台 | **规划导向**：告诉他们"何时开始刷、刷什么、刷多少" |
| 面试作弊 | **能力提升正道**：拒绝作弊，专注长期能力建设 |
| 静态路线图 | **动态个性化Agent**：根据学生情况实时调整 |
| 通用大模型 | **垂直深度 + 长期记忆 + 主动推送** |
| 真人咨询（高价） | **AI规模化**：1/100价格，24/7可用 |

### 3.3 商业模式建议

| 层级 | 功能范围 | 定价建议 |
|------|---------|---------|
| **基础版** | 职业诊断 + 静态路径 + 社区 | 免费（获客） |
| **进阶版** | 动态调整 + 实习/求职管理 + 提醒 | ¥29-49/月 |
| **专业版** | 1对1 AI深度咨询 + 项目评审 + 模拟面试 | ¥99-199/月 |
| **企业版** | 高校/培训机构SaaS批量授权 | B2B定价 |

## 四、系统架构概览

### 4.1 架构哲学：先跑起来，再拆开

**MVP阶段采用「模块化单体」架构**，内部按微服务边界严格模块化，6-12个月后平滑拆分为微服务。

### 4.2 六层架构总览

```
┌────────────────────────────────────────────┐
│  【接入层】CDN + WAF + SLB + Kong API Gateway │
├────────────────────────────────────────────┤
│  【终端层】Web(PC) | H5/APP | 微信小程序        │
├────────────────────────────────────────────┤
│  【应用层】用户 | 对话 | 路径 | 简历 | 面试 | 求职 │
├────────────────────────────────────────────┤
│  【AI中台】Agent编排 | LLM路由 | RAG检索 | 记忆 │
├────────────────────────────────────────────┤
│  【数据中台】PostgreSQL | Milvus | Redis | Kafka│
├────────────────────────────────────────────┤
│  【基础设施】K8s | 阿里云ACK | 监控 | CI/CD      │
└────────────────────────────────────────────┘
```

### 4.3 AI Agent核心架构：Multi-Agent + ReAct

```
              ┌───────────────────────┐
              │     主控Agent（调度器）   │
              │   意图识别 & 任务路由     │
              └───────────┬───────────┘
        ┌────────┬───────┼───────┬────────┐
        ▼        ▼       ▼       ▼        ▼
    ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
    │对话  │ │路径  │ │简历  │ │面试  │ │分析  │
    │Agent │ │Agent │ │Agent │ │Agent │ │Agent │
    └──────┘ └──────┘ └──────┘ └──────┘ └──────┘
        │        │       │       │        │
        └────────┴───────┼───────┴────────┘
                         ▼
              ┌───────────────────────┐
              │  共享基础设施层          │
              │ LLM路由 | RAG | 记忆   │
              └───────────────────────┘
```

**五大子Agent分工**：

| Agent | 职责 | 模型策略 | 专用工具 |
|-------|------|---------|---------|
| 对话Agent | 职业咨询、心理疏导 | DeepSeek-V3 / GPT-4o | 知识库检索、情感分析 |
| 路径Agent | 学习路径生成与调优 | Claude 3.5 / GPT-4o | 技能图谱、资源库 |
| 简历Agent | 简历解析、JD匹配、优化 | GPT-4o / 多模态 | OCR、关键词提取 |
| 面试Agent | 模拟面试、追问、评分 | Claude 3.5（推理强） | 题库、代码判题 |
| 分析Agent | 能力雷达、差距分析、Offer评估 | GPT-4o | 能力模型库、评分算法 |

### 4.4 长期记忆机制（四层架构）

```
瞬时记忆（对话上下文）    → 内存，当前会话
    ↓
短期记忆（24h会话摘要）    → Redis，TTL过期
    ↓
中期记忆（结构化画像）     → PostgreSQL + 向量摘要
    ↓
长期记忆（用户全周期档案）  → PostgreSQL，永久保留
```

### 4.5 RAG三层知识库

| 层级 | 内容 | 更新频率 | 优先级 |
|------|------|---------|--------|
| L1 权威知识 | 技术体系、岗位能力模型、行业标准 | 季度 | 最高 |
| L2 动态知识 | 招聘JD、面经、行业趋势、薪资数据 | 每周 | 中 |
| L3 社区知识 | 脉脉/牛客/知乎从业者经验分享 | 实时 | 辅助 |

### 4.6 核心技术栈选型

| 层级 | 技术选择 | 选型理由 |
|------|---------|---------|
| 前端 | React 18 + Next.js 14 + Taro 3 | 跨端统一，小程序兼容 |
| 后端 | Python 3.11 (FastAPI) + Go 1.22 (Gin) | AI生态+高性能CRUD分离 |
| AI/LLM | LangChain 0.2 + 多模型路由 | 灵活切换DeepSeek/Claude/GPT-4o |
| 向量库 | Milvus 2.3 | 海量向量高性能检索 |
| 数据库 | PostgreSQL 16 + Redis 7 + Kafka 3.6 | 关系数据+缓存+消息队列 |
| 基础设施 | 阿里云ACK + ArgoCD + Prometheus | 云原生弹性伸缩 |

## 六、AI Agent核心工作流

### 6.1 五阶段工作循环

```
用户输入 → [感知] 意图识别 + 情感分析
    ↓
[理解] 调取用户画像 + 对话上下文 + 知识库检索
    ↓
[规划] 任务分解 + Agent路由 + 工具选择
    ↓
[行动] LLM生成 + 工具调用 + 数据存取
    ↓
[反馈] 流式输出 + 记忆更新 + 主动推送判断
```

### 6.2 典型工作流示例：生成学习路径

```
用户: "我想做Java后端，现在只学了基础语法，每天能学3小时，帮我规划一下"

1. 感知：识别意图 = PATH_PLANNING，复杂度 = HIGH
2. 理解：调取画像 → 大三/双非/零基础/目标大厂
3. 规划：路由到路径Agent，需调用技能图谱 + 资源库
4. 行动：
   - 查询Java后端能力模型（L1知识库）
   - 匹配"零基础→大厂"标准路径模板
   - 按每天3小时计算时间分配
   - LLM生成个性化路径
5. 反馈：
   - 流式返回Markdown格式的学期规划
   - 保存路径到用户档案
   - 设置3天后"进度检查"主动提醒
```

### 6.3 主动推送机制

| 触发类型 | 触发条件 | 推送内容示例 |
|---------|---------|------------|
| 时间节点 | 秋招季前2个月 | "秋招即将开始，你的简历优化进度60%，建议本周完成" |
| 进度异常 | 学习路径连续7天未更新 | "你已连续7天未打卡，需要调整计划吗？" |
| 市场变化 | 目标岗位技能要求变化 | "你目标的后端岗位新增'AI应用开发'要求，建议补充" |
| 求职事件 | 面试前24小时 | "明天有XX公司面试，已为你准备3道高频题" |

## 七、数据架构概览

### 7.1 核心实体关系

```
用户(User)
  ├── 用户画像(Profile) —— 基本信息、目标方向、技能标签
  ├── 对话记录(Dialogue) —— 多轮对话、意图标签、情感标记
  ├── 学习路径(Path) —— 路径节点、完成状态、动态调整历史
  ├── 求职记录(JobHunt) —— 投递公司、岗位、状态流转
  ├── 简历版本(Resume) —— 多版本简历、JD匹配记录
  └── 评估记录(Assessment) —— 能力雷达、模拟面试评分
```

### 7.2 关键数据状态机

**学习路径状态**：已创建 → 进行中 → 已暂停 → 已调整 → 已完成  
**求职进度状态**：意向 → 已投递 → 笔试中 → 面试中 → 待OC → 已Offer → 已拒绝  
**路径节点状态**：未开始 → 学习中 → 已完成 → 已跳过

## 八、安全与合规设计

### 8.1 数据安全三级体系

| 数据级别 | 类型 | 保护措施 |
|---------|------|---------|
| 公开级 | 昵称、学校 | 常规访问控制 |
| 敏感级 | 手机号、邮箱 | AES-256加密存储，TLS 1.3传输 |
| 机密级 | 简历内容、测评结果 | 行列级加密，管理员审批留痕 |

### 8.2 AI内容安全三层防护

1. **输入过滤**：敏感词检测 + 提示注入防护
2. **系统提示加固**：知识库强制绑定，限制回答范围
3. **输出审核**：置信度分级 + 事实核查 + 免责声明

### 8.3 合规要点

- 《个人信息保护法》：数据最小收集、用户可控删除（72小时物理清除）
- GDPR预留：数据导出、被遗忘权、跨境传输限制
- 未成年人保护：未满18岁默认最高隐私模式，对话不上传训练
- 生成式AI管理：明确标注AI生成，不给出绝对化承诺

## 十、关键风险与缓解策略

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| **AI幻觉** | 高 | RAG强制绑定 + 置信度分级 + 人工审核飞轮 + 免责声明 |
| **隐私泄露** | 高 | AES-256加密 + 访问控制 + 定期安全审计 + 应急响应 |
| **用户留存** | 中 | 构建"陪伴式"场景：进度追踪、主动推送、成就体系 |
| **竞品跟进** | 中 | 深耕"全链路职业规划"差异化，建立数据壁垒 |
| **LLM成本** | 中 | 模型路由降本（国产模型优先）+ Token配额 + 响应缓存 |
| **合规风险** | 中 | 内容审核机制 + 避免绝对化承诺 + 法律条款完善 |
